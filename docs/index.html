core.ecs module¶

Functions to retrieve status, start or stop ECS tasks.

class core.ecs.ECSManager¶

Bases: object

A class to organize functions related to the ECS cluster. On init, neccessary AWS clients are initiated, ECS cluster looked up by tag, then initial status pushed to frontend.

get_public_ip(taskArns)¶

Retrieves the public IP of the running task.

Returns

public_ip – Public IP of the running task.

Return type

str

start()¶

Starts Cloud service by increasing desired_count to 1. After that, it waits until the task is running, then pushes a status update.

status()¶

Determines ECS task’s status. - If there are no tasks in the cluster, and desired count is 0, the status is ‘off’. - If there are no tasks, but the desired count is not 0, status is ‘startLoading’. - If there is at least 1 task, but the desired count is 0, status is ‘stopLoading’. - If there is at least 1 task, and it’s status is not RUNNING, the status is ‘startLoading’. - If there is at least 1 task, and it’s status is RUNNING, the status is it’s public IP address.

Returns

status – The status of the task according to the rules described above.

Return type

str

stop()¶

Stops Cloud service by decreasing desired_count to 0. After that, it waits until the task is stopped, then pushes a status update.

class core.s3.S3¶

Bases: object

Since the S3 buckets names has to be globally unique, to avoid clashes, they contain a random string. That string is generated deploy time and saved as an SSM parameter. It is retrieved here.
